# Step 50 - 접근성 테스트

타겟 사용자층과 법적 요구사항을 분석하여 접근성 기준과 테스트 도구를 동적으로 결정한다.

접근성 자동화 테스트를 구현한다.

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다.

**접근성 테스트 단계에서 절대로 plan mode를 사용하지 않는다.**

**검증:**
- `.claude/hooks/accessibility-validator.ps1`에서 자동 검증

**검증 실패 시:**
- WCAG 위반 사항 분석
- 접근성 문제 수정 (ARIA 속성, 대체 텍스트, 키보드 탐색 등)
- 검증 통과할 때까지 반복

서브에이전트는 항상 haiku를 사용한다.
